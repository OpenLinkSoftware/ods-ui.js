Product Overview

OpenLink Data Spaces provide a vast feature set which fulfills every cloud system, mobile or desktop client developer's needs. All features are available through the ODS HTTP API as well as the many standard-compliant interfaces ODS exposes to clients.

User content across the Web, both Intranet and Extranet, is growing exponentially, while time, attention, and comprehension of the resulting data streams is at best static, and typically shrinking. Compounding the problem is the fact that the exponential growth of user generated data has resulted in a proliferation of data silos, due to the popularity of Web 2.0 solution patterns.

All of this is happening at a time when individuals and organizations alike are seeking to do more with less, by exploiting the agility-producing powers of "Open Data Access" across enterprise and/or Web domains.

OpenLink Data Spaces bring the data back to where it belongs: to the owner of the data.

Profile Management Manage User Accounts

ODS brings full profile management. Store your personal details and share details according to ODS' powerful ACL rules with other ODS users or any person that can be identified via their WebID. An ODS profile can hold many personal details including online accounts, favorite things, likes and dislikes, areas of interest, product and service wishes, or the social network.

User profiles are exported as FOAF profiles which can be combined with an X.509 certificate to create fully functional WebIDs for each user profile. As with all data in ODS the profile is accessible as linked data through Virtuoso's SPARQL end-point or by dereferencing the personal URI and requesting an appropriate format like Turtle, JSON-LD, or RDF+XML.

ODS supports a variety of registration and login mechanisms including WebID, OpenID, or OAuth-powered services like Facebook or Twitter. Accounts can be created by providing simple password digest information or by connecting them to existing accounts or identifiers from other services.

Social Networking Manage Friend Lists and Share Information

ODS allows to manage your personal social network. Build your list of friends, share information with them, let ODS send pingbacks to inform them of the shared data.

Each ODS user account exposes a full FOAF profile with optional WebID support.

File Management WebDAV With a Vengence

With the Briefcase ODS provides a WebDAV-compliant platform that offers advanced file-sharing capabilities.

In addition to basic WebDAV file management and its complex ACL rule driven security and file-sharing functionality the Briefcase has a wide range of additional feature that make it one of the most powerful file storage systems available.

The Briefcase benefits from Virtuoso's Sponger module to automatically extract file metadata from a wide array of file formats including MS Office, Open Office, Open Document Format, Content Syndication Formats (RSS 1.0, RSS 2.0, Atom, OPML, OCS), (X)HTML, Web Services Contract Files (BPEL, WSDL), XBRL, XBEL, FOAF, iCalendar, hCalendar, vCARD or Google Base Items. This information is available either as pure WebDAV data or as RDF. The latter is, as with all resources in ODS, available by dereferencing the file URIs using content-negotiation or my querying Virtuoso's SPARQL end-point. This also includes the powerful full-text search capabilities of Virtuoso which can be combined with arbitrary complex structured queries.

Briefcase also supports UNIX-style file-permissions as a simple alternative to the powerful ACL rule system.

Briefcase allows to mount other known web storage solutions like Google Drive, Amazon S3, SkyDrive or Dropbox directly into your ODS Briefcase and apply the same powerful ACL rules to them as to any other resource. Through the same mechanism non-physical WebDAV resources like contacts or feed posts can be provided as files in the WebDAV system.

Smart folders allow to browse files by their meta-data. For example the path DAV/home/[username]/Items/Graphics/Type/JPEG lists all the graphic images that are of type JPEG for a given user.

The Briefcase supports version control for any resources using any client supporting the DeltaV core feature set. Historical versions of resources will appear in a separate virtual folder. DeltaV core WebDAV methods are supported for resources for which versioning has been enabled.

Flexible and Open data access are key in the ODS Briefcase. It can be transparently integrated into any WebDAV-compliant environment like MAC OS X Finder, Windows Explorer, or Linux DAV mounts. In addition as with all resources in ODS the data is exported as RDF and accessible through the Briefcase HTTP API.

Calendar Manage Tasks And Events

ODS provides a full calendar system for event and note management. Schedule new events, add attending people, configure reminders, create tasks with priority, due date, attendees, or manage comments.

ODS can subscribe to external calendar resources via CalDAV or by importing ICS files from any WebDAV system or any URL. In the same way calendar events and tasks can be exported or published to any writable URL or any CalDAV resource. The ODS Calendar supports several formats for resource syndication like RSS, Atom, or RDF SIOC.

Events and tasks can be shared with other users through ODS' advanced ACL rule system allowing fine grained privacy settings on any resource.

The ODS Calendar supports synchronization via SyncML through the ODS Briefcase's powerful virtual folder system.

ODS Clients can either manage events by directly accessing the resources via CalDAV, by writing to an ics file which the Calendar instance is subscribed to, or by using the ODS Calendar API.

Address Book Manage Contacts

ODS comes with a fully featured address book manager which in addition to managing contacts exposes these as FOAF resources, allows access via CardDAV, and supports publication and subscription to and from other CardDAV instances or any URL.

The ODS Address Book can import contacts from local or DAV files in several formats including vCard, FOAF, or CSV, or from external LDAP servers.

Like all resources in ODS contacts are exported as SIOC RDF data and made accessible through Virtuoso's SPARQL endpoint or via content-negotiation on the resources' URIs.

The ODS Address Book supports synchronization via SyncML through the ODS Briefcase's powerful virtual folder system.

ODS clients can either manage contacts by directly accessing the resources via CardDAV, by writing to a vcard file which the address book instance is subscribed to, or by using the ODS Address Book API.

E-Mail The shortest road to fully integrated Webmail

ODS integrates a complete and highly scalable E-Mail solution which ranges from simple E-Mail storage and sending to complex smart folder filtering. The ODS mail solution can act as an email client and a server, supporting IMAP, IMAPS, and POP3. It allows to combine and filter the E-Mails from external accounts with the ones directly stored in ODS.

Thanks to the RDF exposure of the resources and powerful SPARQL-based E-Mail filtering the possibilities of virtual sorting and filtering are endless. Smart folders allow to filter mails by embedded WebID certificates, by content or by an arbitrary number of header values. These folders can then be accessed via IMAP through any compliant client allowing for powerful server-side filtering which is consistent between clients and devices.

ODS Mail provides all the tools required to build a fully featured WebMail system or E-Mail filtering proxy.

Manage Bookmarks Weblinks and More

The ODS Bookmark module allows to construct, maintain, and share vast collections of web resource URLs. Bookmarks can be sorted into folders, imported and exported from and to XBEL (Mozilla Firefox, Microsoft Internet Explorer, Mozilla/Netscape, and Opera bookmarks) or Delicious. Publish all bookmarks to Delicious or any writable URL, subscribe to bookmark feeds from Delicious or any URL including WebDAV.

Bookmarks can be tagged, commented, or shared through ODS' powerful ACL rule system. Smart folders allow to filter bookmarks into virtual hierarchies allowing different views onto the same data.

Like with all resources in ODS bookmarks are exposed as RSS, Atom, or RDF feeds which can be accessed by any client or person with the appropriate permissions. These feeds are directly available in the ODS Briefcase (the ODS WebDAV file storage system) in special virtual folders.

ODS provides a complete bookmark HTTP API allowing to perform all tasks surrounding bookmark management.

Feed Aggregator Combine, Split, Sort, Filter

ODS' Feed Manager module allows to combine feeds from different sources in different formats into new feeds, exposing them in a variety of formats including RSS, Atom, OPML, or RDF.

Feeds can be organized into folders and sub-folders or be filtered and combined via smart folders based on their content, URLs, or manually assigned meta-data like tags. Smart folders essentially are stored searches which can be exposed as new feeds.

As with all content ODS exposes feeds and posts as RDF (using the SIOC ontology) to be queried via SPARQL and extended with additional information using the Virtuoso triple-store.

A feed is an xml-based document that contains content, summary information or blog posts with a links to the source. On-line newspapers, magazines, blogs and other web sites produce channels or "feeds" as another information source.

All of the feed managers functionality is provided by the feed manager HTTP API.

Blogging Create Content

ODS features a complete Weblog system which allows each user to setup an arbitrary number of blogs. ODS Weblog supports a variety of content publishing protocols including Blogger, Meta Weblog, or Atom. As with most other ODS content blog entries are syndicated in various formats including RSS, Atom, OPML, or XBEL.

Blog posts can be automatically archived by year, month, and category. A sophisticated comment engine based on the ODS Discussion system allows to manage comment thread on each blog post. Every aspect of the blog system is protected by ODS' sophisticated ACL rule system allowing to grant access to any person or group of person.

The ODS Weblog exposes search requests in any of the supported syndication formats and additionally OpenSearch and GData. Posts can be queried through free text, GData, XQuery, or XPath queries. Making use of ODS' Linked-Data exposure and Virtuoso's SPARQL endpoint any blog can be queried through SPARQL, always controlled by the powerful ACL rule system.

ODS Weblog can send pings (TrackBack) to a variety of third-party services to publish newly written blog posts.

Blog posts can be categorized and tagged including semantic MOAT (Meaning Of A Tag) annotations which connect tags to a semantic meaning.

In addition to the publishing protocols which allow posting of blog entries via third-party tools ODS provides the Weblog HTTP API which allows to control every aspect of a Weblog instance.

Discussions Comment on Anything

The ODS Discussion module adds support for comment threads to all other ODS modules.

Users can add comments to calendar events and tasks, to blog entries, even to bookmarks. As with all resources in ODS the permissions on comments are controlled by sophisticated ACL rules based on people's personal URIs (WebIDs). Each comment can contain file attachments and can be tagged with arbitrary keywords.

The ODS discussion system is powered by Virtuoso's news server implementation which also allows to subscribe to any news feed available. This includes any comment thread on any ODS resource since these are also exposed as news feeds.

Polls Manager Gather Opinions

With the ODS Polls Manager users can create, edit, and vote on polls. Polls can consist of one of more questions with several choices to choose from. Polls can be scheduled for a certain time or be started and closed manually. A poll can be configured to allow only a single or several votes.

Like all content in ODS polls can be shared with other persons of be made public using the powerful ACL rule system ODS provides.

All polls are exported as syndication gems in several formats including RSS, Atom, or SIOC RDF. The ODS Polls HTTP API allows to control each aspect of the polls manager through any client.

ACL Share Information

ODS features comprehensive ACL rules for any kind of data stored in the system. Share information with selected people, with groups of people, even with people whose profiles fulfill certain conditions. Thanks to WebID the people do not require an account with ODS. All they require is to authenticate themselves with their WebID which ODS can map to the individual users' ACL rules.

ODS has a complex ACL rule system which supports fixed groups of people and dynamic groups. Dynamic groups are defined by conditions that the persons need to fulfill. A typical example could be to create the group of all friends by using the foaf:knows relation as condition. Other examples include the exploitation of Virtuoso's transitivity support by creating a family group. All this is possible via dynamic groups defined by arbitrarily complex SPARQL ASK queries.

Standard Compliance WebDAV, CalDAV, CardDAV, RSS, Atom, XBEL, ...

ODS is fully standard-compliant. Data is exposed using established protocols like WebDAV for files, CardDAV for contacts, CalDAV for calendar items, RSS and Atom for feeds, XBEL for Bookmarks, and so on.

As such ODS can be used in different ways by clients. Clients can either use the formerly mentioned standard interfaces to access many of the ODS features which are also part of the standard interfaces. Or clients can use the ODS HTTP API to get access to every last feature ODS has to offer.

  • Access file via WebDAV.
  • Manage your address book via any CardDAV-compliant client.
  • Manage your calendar via any CalDAV-compliant client.
  • Retrieve the structured description of any resource in ODS by resolving its URI.
  • Query data via Virtuoso's SPARQL HTTP end-point.
  • Access syndication feeds on most information in ODS through many formats including RSS, Atom, or OPML.
  • Post new blog entries through established XML-RPC protocols such as Blogger or Meta Weblog.

Linked Data Data Exposed as RDF

ODS provides full linked-data exposure for every piece of information stored within. Any information provided by ODS can be queried through Virtuoso's SPARQL endpoint or be retrieved by dereferencing the resources' URIs and requesting an appropriate encoding of the data. Like everything in ODS the SPARQL endpoint is ACL-protected, only providing the information the requesting person has access to.

Search Query all information in ODS

ODS, being built on top of Virtuoso, takes advantage of the powerful query capabilities. All resources in ODS are exposed as RDF and can thus be queries via SPARQL. This includes full-text matching on text literals. As with all information in ODS the query results are subject to the ACL rules setup by the user.
